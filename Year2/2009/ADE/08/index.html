8. Mergesort & Quicksort

27/02/23

Divide-and-Conquer​

  • General algorithm design paradigm
    • Divide - Divide the input data SSS in two disjoint subsets S1S_1S1​ and S2S_2S2​
    • Recur - Solve the subproblems associated with S1S_1S1​ and S2S_2S2​
    • Conquer - Combine the solutions for S1S_1S1​ and S2S_2S2​ into a solution for SSS
  • Base case is either size 0 or 1

Merge Sort​

  • On an input sequence (array/list) SSS with nnn elements consists of three steps:
    • Divide - Partition SSS into two sequences S1S_1S1​ and S2S_2S2​ of about n/2n/2n/2 elements each
    • Recur - Recursively sort S1S_1S1​ and S2S_2S2​
    • Conquer - Merge S1S_1S1​ and S2S_2S2​ into a unique sorted sequence

Merge-Sort Tree​

  • An execution of merge-sort is depicted by a binary tree
    • each node represents a recursive call of merge-sort and stores
      • unsorted sequences before the execution and its partition
      • sorted sequence at the end of the execution
    • the root is the initial call
    • the leaves are calls on sub sequences of size 0 or 1

Analysis of Merge-Sort​

  • The height hhh of the merge-sort tree is O(log⁡nO(\log nO(logn)
  • The overall amount of work done at all the nodes at depth iii is O(n)O(n)O(n)
    • partition and merge 2i2^i2i sequences of size n/2in/2^in/2i
    • make 2i+12^{i+1}2i+1 recursive calls
    • numbers all occur and are used at each depth
  • Thus, the total running time of the merge sort is O(nlog⁡n)O(n\log n)O(nlogn)

Quick-sort​

  • In mergesort the divide is simple but the merge is complicated. Quicksort replaces the merge part
  • When the lists A and B are sorted and known to be in disjoin ordered ranges
  • If A and B are stored as consecutive sub-arrays, then merge actually needs no work at all, just forget the boundary

3-way split​

  • Quick-sort is a randomised sorting algorithm based on the divide-and-conquer paradigm
    • Divide - pick a random element xxx (pivot) and partition SSS into
      • LLL elements less than xxx
      • EEE elements equal to xxx
      • GGG elements greater than xxx
    • Recur - sort LLL and GGG
    • Conquer - join L,E,GL,E,GL,E,G

2-way split​

  • Divide - pick a random element xxx (pivot) and partition SSS into 
    - $L$ elements less than $x$
    - $GE$ elements greater than or equal to$x$
  • Recur - sort LLL and GEGEGE
  • Conquer - join L,GEL,GEL,GE

Partitions of Lists​

  • If storing L,E,GL,E,GL,E,G as separate structures, we partition an input sequence as follows:
    • Remove, in turn, each element yyy from SSS and
    • Insert yyy into L,E,GL,E,GL,E,G or depending on the result of the comparison with the pivot xxx
  • Each insertion and removal is at the beginning (or end) of the sequence, and hence takes O(1)O(1)O(1) time
  • Thus, the partition step of quick-sort takes O(n)O(n)O(n) time

In-place - Means they only a little extra space is used to store data elements
